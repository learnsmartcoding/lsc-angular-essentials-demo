Deep Dive: The Signals Trio

The foundation of reactivity. It holds a value and notifies Angular when that value changes.

  • set(): Completely replaces the value.
  • update(): Computes a new value based on the previous one.
  • asReadonly(): Protects the signal from being changed elsewhere.
// Syntax
count = signal(0); 
this.count.set(10); 
this.count.update(v => v + 1);

Read-only signals that derive their value from other signals. They are efficient and lazy.

  • Lazy: It only calculates when someone actually reads it.
  • Memoized: It caches the result. If dependencies haven't changed, it returns the cached value.
// Syntax
total = computed(() => this.price() * this.qty());

A side-effect operation that runs automatically whenever its dependent signals change.

  • Use cases: Logging, Analytics, syncing with LocalStorage.
  • Rules: Avoid setting other signals inside an effect (unless allowed). 
// Syntax
effect(() => {
  localStorage.setItem('qty', this.qty());
});
